Curved contrail — technique comparison

Standalone mockup for brainstorming (not wired into the app). budget-app login currently has no background planes; when you add them, these patterns map to the three options discussed: A shared geometry (path + stroke trim to the wake behind the plane), B sampled polyline, C canvas fade. Path is a single illustrative S-curve; production would use your four flight patterns. A uses arc-length math so dots/stroke only appear after the plane has passed (see panel copy).

A — Dotted wake (mask trim + dot pattern)

Mask + dotted stroke

Wake rule: only the subpath immediately behind the plane (last w units of arc length) is stroked. Nothing ahead of the nose, nothing on parts not yet visited.

With path length L and head at distance s from the start, use a compound stroke-dasharray (alternating dash, gap):

  • If s ≥ w: 0, (s−w), w, (L−s) → gap hides [0,s−w), dash draws [s−w, s], gap hides the rest.
  • If s < w: 0, 0, s, (L−s) → trail grows from the start as the plane accelerates onto the path.

Dots: the visible path uses a fixed stroke-dasharray (e.g. 3 14). A <mask> holds a duplicate path with a thick white stroke and the same compound trim as above, so only the wake is opaque; the dotted line shows through there only.

This mockup drives s from one clock (here requestAnimationFrame); in production, drive the mask path’s trim from the same progress as offset-distance × L.

  • Pure CSS: animate mask stroke dash + plane offset-distance in lockstep (verbose).
  • Widen stroke-width on the mask path if dots look clipped at turns.

B — RAF sample → polyline

requestAnimationFrame

JavaScript advances t along the same Bezier, pushes screen points into a ring buffer, draws a dotted polyline.

  • Matches any motion source (CSS or JS)
  • ~15 planes × sampling cost
  • Trail follows actual pixels

C — Canvas + fade buffer

alpha clear

Each frame: fill with semi-transparent background to decay old pixels, stroke new segment. Soft “smoke” tail.

  • Organic fade; great for glow
  • Day/night theming = stroke/fill colors
  • More tuning (fade rate, blur)
